# Phase 8: Micro-Scale Crowd Computing Design
## Architecture for Deep Parametric Sweeps & Monte-Carlo Validation

This document outlines the architectural design for Phase 8. With the successful completion of the V1.0 "Shallow Sweep" which confirmed macroscopic topological anomalies, we now transition to the distributed "Deep Sweep."

Our objective is twofold:
1. **Find the exact geometric shape of the K3 knot** by testing billions of geometric permutations.
2. **Establish a 5-sigma statistical proof** that our discoveries are not random noise.

---

## 1. Deep Parametric Sweep Work-Unit Generator

The goal of the Deep Sweep is to take a known anomaly (identified during the Shallow Sweep) and run a multidimensional parameter scan over it to find the precise alignment of the $S_{1,2}$ topological knot.

### 1.1 The Parameter Space
The Picard-Fuchs topological template will no longer be static. It will be allowed 6 to 8 degrees of freedom:
*   **Translation ($x, y, z$):** Micro-adjustments around the discovered centroid.
*   **Rotation ($\theta, \phi, \psi$):** 3D Euler angles to align the knot with the cosmic filament.
*   **Scale ($s$):** Expanding or contracting the topological manifold.
*   **Deformation ($d_{12}, d_{21}$):** Tension parameters affecting the symmetry breaking ratio.

### 1.2 The Spawner Architecture (Backend)
1.  **Anomaly Targeting:** The system reads `discoveries.json` and selects a high-priority "High Gravity Hub".
2.  **Permutation Matrix Generation:** The `deep_sweep_generator.py` script calculates the bounds for the parameter space (e.g., $360^\circ$ rotation divided into $0.1^\circ$ increments). This yields billions of possible permutation matrices.
3.  **Work-Unit Chunking:** The parameter space is divided into manageable "Chunks." Each chunk contains the static raw galaxy data for that sector, PLUS an array of 10,000 to 50,000 permutation matrices to test.

### 1.3 Client Execution (BOINC / Browser Nodes)
1.  The client node downloads the chunk.
2.  For each permutation matrix in the chunk, the client:
    *   Applies the transformation to the theoretical Picard-Fuchs template.
    *   Calculates the topological fit ($\beta_0, \beta_1$ Betti numbers, $\Delta$ index) against the local galaxies.
3.  The client keeps only the "Top 5" highest-scoring fits and discards the rest.
4.  The client returns a lightweight receipt containing the winning parameter matrices and their scores.

### 1.4 Aggregation (The Map-Reduce)
The Cloud Run dispatcher acts as the reducer. As millions of clients return their top scores, the backend builds a multi-dimensional gradient descent map, converging precisely on the true microscopic shape of the K3 manifold.

## 2. Monte-Carlo Noise Generator

To publish these findings in high-tier physics journals, we must achieve a **5-sigma level of confidence ($p < 3 \times 10^{-7}$)**. We must prove the K3 signature does not arise by chance.

### 2.1 Synthetic Universe Spawner (Backend)
We will build `tools/monte_carlo_spawner.py`. This generator will produce randomized, synthetic cosmic sectors that mimic the macroscopic properties of the real universe without any underlying topological rules:
*   **Poisson Point Processes:** Generates a uniformly random distribution of points (galaxies).
*   **Gaussian Random Fields:** Simulates standard $\Lambda$CDM clustering (clumping) without introducing complex K3 manifolds.

### 2.2 Blind Dispatching (The Double-Blind Trial)
1.  The synthetic universes are sliced into standard 500KB JSON chunks, identical in format to real SDSS data.
2.  The backend assigns these "Control Chunks" to the citizen network alongside "Real Chunks". 
3.  **Crucially:** The client nodes (browsers/BOINC) *do not know* if they are processing real data or Monte-Carlo noise.

### 2.3 The 5-Sigma Verification Ledger
1.  As the network processes the noise, the backend tracks the highest possible asymmetry scores generated by the Monte-Carlo data. This establishes the **Baseline Noise Floor**.
2.  By comparing the $\Delta$ peaks from the real SDSS data against the distribution curve of the noise floor, the backend calculates the standard deviation ($\sigma$).
3.  When the real data consistently yields $\Delta$ peaks $> 5\sigma$ above the noise floor, the structural reality of the S1,2 symmetry breaking is statistically proven.
