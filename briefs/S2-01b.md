## Context

**Why this matters:** K3_CRITERIA.md criterion **C3b** — the Shioda-Inose moduli map F — is the geometric relation that establishes whether an order-3 Picard-Fuchs operator truly represents the periods of a K3 surface (the claimed "base" of the F-theory compactification). Without C3b verification, the candidate selection has no geometric grounding; S3-00 has no certificate to build on.

**What it does:** Given a pair of Picard-Fuchs operators (one order-2 via the elliptic curve, one order-3 via the K3), the checker constructs and verifies the Shioda-Inose correspondence — the algebraic relation F(z_order2; z_order3) whose evaluation links the two sequences. It is a **mechanical, deterministic check**: no numerical approximation, no LLM guessing, pure symbolic algebra.

**Tier and routing rule (verifier-first):** T1 writes the code; T2 writes the pytest suite; Sympy/Sage are the verifiers (exact symbolic algebra, no floating point). CI rule: commit passes iff all golden tests pass deterministically.

## Specification

### Inputs

1. **Two recurrence relations:**
   - Order-2 recurrence (elliptic curve, S₁₂ or S₂₁ from literature)
   - Order-3 recurrence (K3 candidate: s₇ or s₁₀, from certified `cooper_s10_kernel.py` or `cooper_exact.py`)
   - Each recurrence provided as integer coefficients and initial conditions

2. **Target candidate:** Named pair (e.g., `(s10, S12)` or `(s7, S21)`)

3. **Expansion order N:** Symbolic verification to N terms (default N=50, configurable up to 200 if needed)

4. **Hauptmodul data (input from literature):** For each candidate pair, the first principal modulus (q-expansion coefficients or algebraic expression) — these are not computed by the checker, but *input* from verified literature sources (cited in K3_CRITERIA.md C3b section)

### Output

1. **Success case (PASS):**
   - F(z_elliptic) as an explicit algebraic or q-series relation in terms of z_order3
   - Verification status: "PASS(N)" if both order-2 and order-3 expansions agree on both sides of F(z_e) = F_canonical to order N with zero mismatches
   - Determinism certificate: SHA256 hash of input recurrences + checker code, guaranteeing reproducibility
   - Assumption tags: `[A-ONT]` (the F relation is geometric, contingent on compactification realization)

2. **Failure case (FAIL):**
   - No valid F exists (coefficient mismatch persists beyond numerical tolerance)
   - Output: "FAIL — no Shioda-Inose map found for <pair>"
   - Assumption tags: `[A-ONT]` still tagged (failure of the compactification realization hypothesis)
   - Consequence: candidate pair removed from selection via F1 branch (K3_SELECTION_REPORT.md)

3. **Ambiguous case (CONDITIONAL):**
   - Multiple candidate F relations exist at order N; expansion must continue to order 2N to disambiguate
   - Output: "CONDITIONAL(N, N_required=2N) — multiple F candidates at order N; re-run with N=2N"

### Algorithm Sketch (for briefing T1)

1. **Extract Picard-Fuchs operators:** From the two recurrences, compute the formal power-series solutions Π₀^(order-2)(z_e) and Π₀^(order-3)(z_b) to order N.
   
2. **Hauptmodul mapping:** For each order, the periods Π₀ are related to the Hauptmodul λ(q) by a linear relation: Π₀(z) = A + B·λ(q(z)). The checker retrieves the known (A, B, λ_series) from K3_CRITERIA.md's "reference data" section for each candidate.

3. **Construct F:** The Shioda-Inose map F: z_e → z_b is implicit in the statement that both periods' Hauptmoduli are *the same function* of a shared modular parameter q. Algebraically:
   - Express q as a function of z_e via the order-2 relation
   - Express z_b as a function of q via the order-3 relation
   - Compose: z_b = F(z_e) = z_b(q(z_e))

4. **Verify:** Expand both sides to order N and confirm coefficient-wise equality. Compute margin (max absolute mismatch across all N terms).

5. **Output certificate:** JSON dict with fields: `{"status": "PASS", "order": N, "margin": <max_error>, "F_series": <q_series>, "determinism_hash": <sha>, "assumptions": ["A-ONT"]}`

### Golden Test Cases

**A. Known-good pair (example: s₇ with S₁₂):**
- Input: exact recurrence coefficients from literature (Cooper 2012; AESZ)
- Expected output: `PASS(50)` with F as a polynomial/q-expansion relation matching the canonical Shioda-Inose form from the literature (cite the source)
- Margin: <10⁻¹⁵ (machine epsilon for exact rational arithmetic)

**B. Known-bad pair (example: s₁₀ with a generic order-2 sequence):**
- Input: s₁₀ with a made-up order-2 recurrence (not from any compactification)
- Expected output: `FAIL — no Shioda-Inose map...` or `CONDITIONAL` if partial agreement to low order, but failure to disambiguate at 2N
- Purpose: confirm the checker does not hallucinate F relations

**C. Stress test (high N):**
- Input: a known-good pair, N=200
- Expected output: `PASS(200)` with margin stable to machine precision
- Purpose: confirm no numerical drift; margin should not grow with N

**D. Symmetry check (optional but good):**
- Input: test whether swapping order-2 and order-3 breaks the map (it should, since they are geometrically distinct)
- Expected output: `FAIL — swap breaks relation`
