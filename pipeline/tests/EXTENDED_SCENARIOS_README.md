## Test Classes

### 1. **TestClosureMultipleSNR** — Signal Recovery Across SNR Levels

Tests closure property (injected signal must be recovered) across amplitude range.

| Test | Amplitudes | Check |
|------|-----------|-------|
| `test_signal_recovery_snr_sweep` | 0.5, 1.0, 2.0, 3.0, 4.0 | Recovery at each level |
| `test_snr_monotonicity` | All above | Statistic ↑, p-value ↓ |

**Rationale:** Ensures pipeline correctly scales with signal strength. Monotonicity is fundamental to scientific inference.

---

### 2. **TestNullMultipleAlpha** — False-Positive Rate Control

Tests that false-positive rate is calibrated across different alpha levels (significance thresholds).

| Test | Alpha Levels | Check |
|------|----------|-------|
| `test_fp_rate_at_alpha` | 0.01, 0.05, 0.10 | FP rate ≤ alpha + 3σ |

**Rationale:** Different analyses may use different significance levels. Pipeline must calibrate correctly at each.

---

### 3. **TestRobustnessRepeatability** — Determinism and Seed Robustness

Tests reproducibility (same seed → same result) and robustness (different seeds → consistent behavior).

| Test | Coverage |
|------|----------|
| `test_deterministic_result_same_seed` | Identical results with identical seeds |
| `test_robust_across_null_seeds` | Signal recovery holds for 3 different null distributions |

**Rationale:** Reproducibility is non-negotiable. Robustness across null-generation seeds verifies the signal signature is robust.

---

### 4. **TestFieldSizeScaling** — Scaling with Dimensionality

Tests pipeline correctness as field size N changes (8, 16, 24, 32).

| Test | Coverage |
|------|----------|
| `test_signal_recovery_across_sizes` | Signal recovery at each N |
| `test_null_calibration_across_sizes` | False-positive rate controlled at each N |

**Rationale:** S3-02 pipeline will work on different grid resolutions. Must behave consistently.

---

### 5. **TestEdgeCases** — Boundary Conditions

Tests near-threshold and extreme signal amplitudes.

| Test | Scenario |
|------|----------|
| `test_weak_signal_at_detection_threshold` | amp=0.3 (weak, low power) |
| `test_zero_signal_is_null` | amp=0.0 (no signal) |
| `test_very_strong_signal_always_rejects` | amp=10.0 (extremely strong) |

**Rationale:** Edge cases often reveal bugs. Strong signal must always reject; zero signal must not always reject (only at α rate).
